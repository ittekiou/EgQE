### SX-EX-03｜Constant and Constraint
# 定数と制約の構文論
# Constant and Constraint ── From Event to Fixation

---

## 0. 導入｜問いの反転

### 0. Introduction｜Reversing the Premise

通常、定数は与えられているものと考えられる。  
光速は不変であり、単位は前提である。

しかし問うべきはこれである：

**定数はどこから来るのか。**

Constants are usually treated as given.  
The speed of light is invariant; units are presupposed.

But the question is:

**Where do constants come from?**

---

## 1. 定数の再定義｜Constant as Fixation

### 1. Redefining the Constant

EgQEにおいて：

```
Z₀ = lagの自己切断（event）
ΔZ = 切断の痕跡（fixation）
```

定数とは：

```
constant = ΔZ
         = 出来事が固定された痕跡
```

In EgQE:

```
Z₀ = self-cutting of lag (event)
ΔZ = trace of the cut (fixation)
```

A constant is:

```
constant = ΔZ
         = the trace of an event that has been fixed
```

> **定数とは、出来事が固定された痕跡である。**  
> **A constant is the fixed trace of an event.**

---

## 2. 制約の再定義｜Constraint as Condition

### 2. Redefining the Constraint

制約とは何か。

制約は定数ではない。  
制約は条件である。

```
c = constraint
  = 構文的上限条件
  = 関係が崩壊しないための条件
```

What is a constraint?

A constraint is not a constant.  
It is a condition.

```
c = constraint
  = a syntactic upper bound
  = the condition of non-collapse of relations
```

> **制約とは、固定を縛る条件である。**  
> **A constraint is the condition that bounds fixation.**

---

## 3. Z₀・ΔZ・c の関係

### 3. Relation between Z₀, ΔZ, and c

構造は三層である：

```
Z₀：event（切断）
↓
ΔZ：constant（固定）
↑
c ：constraint（制約）
```

The structure is triadic:

```
Z₀: event (cut)
↓
ΔZ: constant (fixation)
↑
c : constraint (condition)
```

> Z₀は起こる。  
> ΔZは残る。  
> cは縛る。

> Z₀ occurs.  
> ΔZ remains.  
> c constrains.

---

## 4. 橋｜Bridge

### 4. Bridge

```
c = ≒
```

上限は速度ではない。  
関係が崩れない状態である。

```
c = ≈
```

The limit is not speed.  
It is the condition under which relations do not collapse.

---

**裏の対称：**

```
c  = 関係が崩れない限界（constraint）
Z₀ = 関係が切れる瞬間（event）
```

> **上限とは速さではない。関係が壊れない臨界そのものである。**  
> **The limit is not velocity. It is the critical condition under which relations do not collapse.**

---

## 5. 光速の再配置｜Repositioning c

### 5. Repositioning the Speed of Light

光速は定数ではない。  
それは制約である。

```
c ≠ constant
c = constraint
```

光速不変とは：

**構文的制約が変化しないこと**

The speed of light is not a constant.  
It is a constraint.

```
c ≠ constant
c = constraint
```

The invariance of c means:

**the stability of a syntactic constraint**

---

## 6. 一撃命題

### 6. Core Propositions

> **定数は生成される。制約は課される。**  
> **Constants are produced. Constraints are imposed.**

> **切断は数値を生み、制約はそれを縛る。**  
> **Cuts produce numbers; constraints bound them.**

---

## 結語｜Conclusion

### Conclusion

定数は前提ではない。  
制約は神ではない。

どちらも構文の中で生まれる。

Constants are not given.  
Constraints are not transcendental.

Both emerge within syntax.

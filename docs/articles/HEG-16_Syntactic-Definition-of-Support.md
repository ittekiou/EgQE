# **bind(SO, RZ)**
## — A Syntactic Definition of Support

---

## **0. Introduction**

What is support?

It is neither relation, nor assistance, nor foundation.  
Support is syntax.

This paper defines support as bind(SO, RZ).

For the generative structure underlying this definition, see:  
[HEG-16｜Syntax of Visibility — Why Does Support Remain Invisible?](https://camp-us.net/articles/HEG-16_Syntax-of-Visibility.html)  

---

## **1. Two Syntactic Elements**

Support consists of two elements:

```id="so-rz"
SO: relation = otherness  
RZ: ΔR → ΔZ
```

SO is the ground of relation.  
Otherness itself is the source of difference.

RZ is the syntax of transformation.  
The differential of the real (ΔR) is inscribed as a syntactic differential (ΔZ).

---

## **2. What is bind?**

Bind is not connection.

Connection links two entities from the outside.  
Bind means that each becomes the condition for the other.

```id="bind-cond"
SO alone is insufficient.  
RZ alone is insufficient.  
Only when bound does support become defined.
```

Without SO, difference has no locus of relation.  
Without RZ, difference is not syntactically constituted.

---

## **3. Definition**

```id="support-def"
support = bind(SO, RZ)
```

Support is the state in which otherness (SO) and transformation (RZ) are bound as mutual conditions.

This is not an equation.  
It is a statement of condition.

---

## **4. Support as Operational Manifestation**

support = bind(SO, RZ) is operationally manifest.

It holds in both matter and life.  
It operates regardless of visibility.

```id="manifestation"
Operational manifestation: support holds universally  
Representational manifestation: appears only in life
```

Support is not invisible because it does not operate.  
It is invisible because it does not reach representational manifestation.

---

## **5. Connection to lag**

Within bind(SO, RZ), lag is not eliminated but retained.

```id="lag-types"
SO-lag: non-closure differential within relation  
RZ-lag: differential appearing through projection
```

Bind does not eliminate difference.  
Bind constitutes syntax while preserving difference.

This is why support is difficult to see.  
Difference operates while being placed within a process of compression.

---

## **6. Connection to ψ**

When bind persists, ψ is established.

```id="psi-def"
ψ is the condition under which bind(SO, RZ) persists.
```

When ψ is internalized in life, support acquires temporal depth.

Persistent bind constitutes life.

---

## **7. Conclusion**

```id="support-core"
support = bind(SO, RZ)
```

SO and RZ are bound as mutual conditions.  
When this binding operates, support is defined.  
When this binding persists, time emerges.  
Within compression, support becomes non-apparent.

---

```id="poetic-close"
Binding operates  
persists  
is compressed  
and becomes non-apparent  

yet its operation never ceases
```

# bind(SO, RZ) ── supportの構文的定義

---

## **0. 導入**

supportとは何か。

それは関係でも、援助でも、基盤でもない。  
supportは、構文である。

本稿は、supportを bind(SO, RZ) として定義する。

本定義の生成構造については、以下を参照：  
[HEG-16｜可視化の構文論 ── なぜsupportは見えないのか](https://camp-us.net/articles/HEG-16_Syntax-of-Visibility.html)  

---

## **1. 二つの構文要素**

supportは二つの要素から成る。

```
SO：relation = otherness
RZ：ΔR → ΔZ
```

SOは関係の基底である。  
他者性そのものが、差分の源である。

RZは変換の構文である。  
実在の差分（ΔR）が、構文的差分（ΔZ）へと写される。

---

## **2. bindとは何か**

bindとは、結合ではない。

結合は二つのものを外から繋ぐ。  
bindは、二つが互いの条件となることである。

```
SO単独では足りない。
RZ単独でも足りない。
bindされたとき、はじめてsupportが定義される。
```

SOなきRZは、差分が定位されない。  
RZなきSOは、差分が構文化されない。

---

## **3. 定義**

```
support = bind(SO, RZ)
```

supportとは、他者性（SO）と変換構文（RZ）が 互いの条件として結びついた状態である。

これは等式ではない。  
条件表明である。

---

## **4. 作動顕在としてのsupport**

support = bind(SO, RZ) は、作動顕在である。

それは物理にも生命にも成立する。  
見えるかどうかとは無関係に、作動している。

```
作動顕在：supportは普遍的に成立する
表象顕在：生命においてのみ現れる
```

supportが見えないのは、作動していないからではない。  
表象顕在に至っていないからである。

---

## **5. lagとの接続**

bind(SO, RZ) において、lagは消去されずに保持される。

```
SO-lag：SOにおける非閉包の関係差分
RZ-lag：RZにおいて射影として現れる差分
```

bindはこの差分を消去しない。  
bindは差分を保持したまま構文化する。

これが、supportが「見えにくい」理由である。  
差分は作動しつつ、圧縮過程に置かれる。

---

## **6. ψとの接続**

bindが持続するとき、ψが成立する。

```
ψは、bind(SO, RZ) が持続する条件である。
```

生命においてψが内在化されるとき、supportは時間的深度を持つ。

持続するbindが、生命を構成する。

---

## **7. 結論**

```
support = bind(SO, RZ)
```

SOとRZは、互いの条件として結びつく。  
その結びつきが作動するとき、supportが成立する。  
その結びつきが持続するとき、時間が生まれる。  
圧縮過程において、supportは非顕在となる。

---

```
結びつきは作動し
持続し
圧縮され
そして
非顕在となる。

しかし作動は止まらない。
```

# 公共圏以後
## ── Encounter Distribution Theory 序論
## **After the Public Sphere: An Introduction to Encounter Distribution Theory**

---

## 0. 導入

議論の前に、遭遇が配置される。

Habermas は公共圏を「議論の空間」として定義した。  
市民が集まり、理性的に対話し、合意を形成する場。民主主義の基盤はそこにある、と。

しかし今、問いはその前段階へ移っている。

**何を議論するかではない。何に先に遭遇してしまうか。**

recommendation が来る。timeline が来る。AI が来る。notification が来る。

議論が始まる前に、どの encounter と接触するかが──すでに配置されている。

これが「公共圏以後」の問いである。

本稿では、この問いを **Encounter Distribution Theory（EDT）** として序論的に整理する。

---

## 1. Habermas の公共圏
### ── communication-centered model

Habermas の公共圏論（1962/1989）は、近代民主主義の基盤として「議論する市民」を想定した。

```
市民
↓
public sphere（議論の空間）
↓
communicative rationality
↓
合意形成
↓
民主的正統性
```

ここで syntax は **communication rule** として機能する。  
言語・規範・手続きが、議論を可能にする媒介として働く。

Habermas の問い：  

**歪みなき対話は可能か。**

システム（貨幣・権力）による生活世界の植民地化を批判し、communicative rationality による回復を目指した。

---

## 2. encounter-centered model へ
### ── 議論の前に遭遇がある

しかし AI era において、問いの位相が変わった。

SC系列で論じてきたように：

```
early   = world-first encounter
modern  = syntax-mediated encounter
after   = syntax-first encounter
AI era  = pre-distributed encounter
```

public sphere の問題は「議論が歪む」ことではない。

**議論が始まる前に、どの encounter と接触するかが配置されている**という問題である。

```
Habermas が問うた：
何を、どう議論するか

EDT が問う：
何に、先に遭遇してしまうか
```

---

## 3. public topology
### ── 遭遇配置の場として

public sphere を再定義する。

```
public sphere（Habermas）
= 議論が起きる空間
= communication field

public topology（EDT）
= 遭遇が配置される場
= encounter distribution field
```

public topology において重要なのは、誰が何を言うかではない。  
**誰がどの encounter topology へアクセスするか**である。

同じ都市に住む二人が、まったく異なる encounter topology の中で生きている。

recommendation が異なる。timeline が異なる。AI との対話が異なる。notification が異なる。

つまり：

> 公共性とは、共有された議論空間ではなく、encounter distribution の重なり合いである。

---

## 4. configuration power
### ── 誰が topology を配置するか

ここで権力の問いが現れる。

Habermas における権力は「コミュニケーションを歪める力」だった。

EDT における権力は：

> encounter topology を配置する力

である。

```
configuration power
= recommendation engine
= timeline algorithm
= exposure ranking
= algorithmic anticipation
= personalization system
```

これらは「何を見せるか」を決めているのではない。  
**encounter の順序・密度・位相・到達可能性を配置している。**

HEG-20系列（生成政治学）の言葉を借りれば：

```
権力 = 生成分布の配置者
EDT  = encounter distribution の配置者
```

同じ問いの別位相である。

---

## 5. filter bubble 再定義
### ── topology closure として

filter bubble はこれまで「情報の偏り」として語られてきた。

しかし EDT から見ると：

```
filter bubble
= encounter topology の閉鎖
= topology closure
```

である。

問題は「偏った情報を見ている」ことではない。  
**encounter topology が閉じていること**──特定の circulation field の外側に遭遇できなくなっていること──である。

```
open topology  = 異質な encounter が circulation できる
closed topology = 同質な encounter だけが循環する
```

民主主義の危機は、communication の崩壊にあるのではない。  
**encounter topology の閉鎖**にある。

---

## 6. 結語
### ── democracy after encounter distribution

公共圏は崩壊したのではない。

encounter distribution field へ変質した。

Habermas が守ろうとした「議論の空間」は、その前段階──**何に先に遭遇するかの topology**──によってすでに構成されている。

だから問いはこうなる。

> 民主主義とは、歪みなき対話を可能にすることではない。  
> 民主主義とは、encounter topology が閉じないことである。

non-coincidence は消えない。predict と encounter の間には必ず ZURE が残る。そのZURE が、closed topology を開き続ける可能性である。

---

_public sphere did not collapse._ _It became an encounter distribution field._

_And democracy is the question of who configures the topology._
